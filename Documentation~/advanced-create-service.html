Create Service Tool

Visual interface for creating services without writing code

Tool Overview

The Create Service Tool provides a streamlined workflow for creating services in Unity. Access it through GAOS > Service Locator > Create New Service in Unity's menu bar.

Two-State Workflow

  • Creation State: Configure service settings and add to review queue
  • Review State: Review, edit, or remove pending services before writing to disk

Service Creation Interface with all configuration options

Service Configuration

Service Settings

  • Service Name: The name of your service class
  • Namespace: Optional namespace for your service (can use existing or create new)
  • Description: Optional description for documentation

Interface Settings

  • Interface Only: Create just the interface without implementation
  • Use Existing Interface: Select an existing interface instead of creating a new one

Implementation Settings

  • Service Type:
    • Regular: Standard C# class service
    • MonoBehaviour: Component-based service for scene functionality
    • ScriptableObject: Asset-based service for configuration and shared data
  • Auto Register:
    • Adds [Service] attribute for automatic registration
    • Configures service name, lifetime, and context
    • For ScriptableObject, adds [CreateAssetMenu] attribute
  • Registration Context:
    • Runtime: Service available only during play mode
    • Editor: Service available only in editor
    • Both: Service available in both contexts
  • Service Lifetime:
    • Singleton: One instance shared across the application
    • Transient: New instance created per request (not available for MonoBehaviour/ScriptableObject)

Directory Settings

  • Scripts Directory: Choose where to save the generated service files
  • Browse: Use the Browse button to select a directory in your project

Bulk Creation Workflow

Create multiple services efficiently in a single session:

Review Interface showing pending services

  1. Configure Service

    Use the creation interface to set up your service details, including name, type, and settings

  2. Add to Queue

    Click "Add Service" to add the configured service to the review queue for bulk processing

  3. Add More Services

    Continue adding more services to create multiple services in one session

  4. Enter Review Mode

    Click "Review Pending Services" to see all services queued for creation

  5. Review and Adjust

    Review each service, make any necessary edits, or remove services from the queue

  6. Generate Services

    Click "Confirm Writing" to generate all service files in your project

Important Notes

  • Service names must be valid C# identifiers
  • The tool validates service configurations before allowing creation
  • Existing files will prompt for confirmation before overwriting
  • Service state is preserved if you accidentally close the window

Best Practices

  • Use descriptive service names that reflect their purpose
  • Organize services in appropriate namespaces for better code organization
  • Consider using the bulk creation workflow for related services
  • Review generated services in the Type Cache Inspector after creation

ScriptableObject Service Tips

  • Initialize services in OnEnable
  • Implement IServiceDisposable for cleanup if needed
  • Consider thread safety for shared data
  • Use SerializeField for Unity Inspector configuration
